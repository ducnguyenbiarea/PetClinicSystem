DTOs Annotation Documentation

Last updated: June 2025

1. Overview of DTO Annotations

DTOs (Data Transfer Objects) in this project use a combination of Lombok, Jackson, and validation annotations to simplify code, control JSON serialization, and enforce data integrity.
Unlike model entities, DTOs do not use JPA annotations and are not mapped to database tables.

2. Common Annotations

Annotations Used in All DTO Classes
Annotation Description
@Getter, @Setter Generates getter and setter methods for all fields (Lombok).
@AllArgsConstructor, @NoArgsConstructor Generates constructors with all arguments and with no arguments (Lombok).
@FieldDefaults(level = AccessLevel.PRIVATE) Sets the default access level for fields to private (Lombok).
@JsonProperty Specifies the JSON property name for a field, and can control access (Jackson).
@JsonPropertyOrder Defines the order of properties in the serialized JSON output (Jackson).
@Size, @NotNull, etc. Validation annotations to enforce constraints on DTO fields (Jakarta Validation).
@Data Shortcut for @Getter, @Setter, @ToString, @EqualsAndHashCode, and @RequiredArgsConstructor (Lombok, used in sub DTOs).

4. Summary

DTO annotations focus on serialization, validation, and code reduction, ensuring that API data is well-structured, secure, and easy to maintain.
