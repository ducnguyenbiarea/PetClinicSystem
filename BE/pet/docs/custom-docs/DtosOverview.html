Pet Clinic DTOs Documentation

Last updated: June 2025

1. Overview

The dtos package defines Data Transfer Objects (DTOs) for the Pet Clinic application. DTOs are used to transfer data between the backend and frontend, or between different layers of the application, without exposing the internal model structure.
DTOs often mirror the structure of model entities but may omit sensitive fields, flatten relationships, or adapt data for API requirements.

2. Common Features

All DTOs typically include:

  • Fields corresponding to model attributes, with possible renaming or type adaptation.
  • Jackson annotations (@JsonProperty, @JsonPropertyOrder) for JSON serialization.
  • Lombok annotations for boilerplate reduction.
  • Static fromEntity() methods to convert from model entities to DTOs.

4. Summary

DTOs provide a safe and efficient way to transfer data in the Pet Clinic system, decoupling API contracts from internal models and supporting validation, serialization, and transformation needs.
